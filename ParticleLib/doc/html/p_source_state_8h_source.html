14/// These functions set the current state needed by Source() and Vertex() actions.
15///
16/// These calls dictate the properties of particles to be created by Source() or Vertex().
17/// When particles are created within a NewActionList() / EndActionList() block, they will
18/// receive attributes from the state that was current when the action list was created (unlike OpenGL).
19/// When in immediate mode (not creating or calling an action list), particles are created
20/// with attributes from the current state.

80 /// Specify the domain for colors and alpha value of new particles.
81 ///
82 /// Your application can interpret the color triple in any color space you choose. RGB is the most common, with colors ranging on 0.0 -> 1.0.
83 /// For example, the PDLine(pVec(1, 0, 0), pVec(1, 1, 0)) will choose points on a line between red and yellow. Points outside the 0.0 -> 1.0
84 /// range will not be clamped by the Particle System API. Some renderers may use colors on the range 0 -> 255, so the domain used to choose
85 /// the colors can be on that range. The alpha value is usually used for transparency.
86 ///
87 /// The particle color does not necessarily need to be used to represent color. It can be interpreted as an arbitrary three-vector.
88 ///
89 /// The default color is 1,1,1,1 (opaque white).
90 PINLINE void Color(const pDomain& cdom) ///< The color domain.
91 {
92 Color_ = std::shared_ptr<pDomain>(cdom.copy());
93 Alpha_ = std::shared_ptr<pDomain>(new PDPoint(pVec(1)));
94 }

128 /// Specify the domain for the size of particles to be created.
129 ///
130 /// All new particles will have a size chosen randomly from within the specified domain. The size values may be negative.
131 ///
132 /// The size is not mass. It does not affect any particle dynamics, including acceleration and bouncing. It is merely a triple of rendering
133 /// attributes, like color, and can be interpreted at the whim of the application programmer (that's you). In particular, the three
134 /// components do not need to be used together as three dimensions of the particle's size. For example, one could be interpreted as radius,
135 /// another as length, and another as density.
136 ///
137 /// The default size is 1,1,1.
138 PINLINE void Size(const pDomain& dom) { Size_ = dom.copy(); }
139

158 /// Specify the initial age of particles to be created.
159 ///
160 /// The age parameter can be positive, zero, or negative. Giving particles different starting ages allows KillOld() to distinguish between
161 /// which to kill in interesting ways. Setting sigma to a non-zero value will give the particles an initial age with a normal distribution
162 /// with mean age and standard deviation sigma. When many particles are created at once this allows a few particles to die at each time step,
163 /// yielding a more natural effect.
164 ///
165 /// The default age is 0 and its sigma is 0.
166 PINLINE void StartingAge(const float age, ///< mean starting age of particles
